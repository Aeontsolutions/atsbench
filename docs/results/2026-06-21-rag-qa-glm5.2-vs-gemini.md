# RAG Q&A benchmark — GLM 5.2 vs Gemini 2.5 Pro (2026-06-21)

Second real `atsbench` run. Workflow: **grounded financial Q&A** (frozen single-turn,
isolated answer generation). Logs are gitignored; this file is the durable record.

## Setup

- **Task:** answer a JSE financial question using only a frozen context snippet, via the team's
  verbatim production "JSE Financial Analyst" synthesis prompt. Single-turn, isolated generation.
- **Dataset:** 13 hand-built records (10 positive financial Qs, 3 negative decline/redirect cases),
  figures fact-checked against public sources (`fixtures/rag_qa/dataset_factcheck.md`).
- **Scoring:** LLM-as-judge = **Claude Sonnet 4.6** (neutral to both candidates, $3/$15 per M),
  3-dim rubric (groundedness / factfulness / goal_completion). **Headline = factfulness fraction**
  (expected_facts satisfied). Judge tokens are *excluded* from the candidate's cost (verified).
- **Routing:** OpenRouter, reasoning enabled, temperature 0.

## Result

| Rank | Model | Factfulness | p50 | p95 | $/answer | Gate | Pareto |
|---|---|---|---|---|---|---|---|
| 1 | **GLM 5.2** | **1.000** | 12.0 s | 42.7 s | **$0.0018** | pass | ★ |
| 2 | Gemini 2.5 Pro | 0.923 ±.052 | 13.2 s | 20.3 s | $0.0100 | pass | ★ |

Secondary (judge dims, 1–5): GLM groundedness **4.77** / goal **5.00**; Gemini **4.46** / **4.77**.

## Verdict

**On grounded financial Q&A, GLM 5.2 beat Gemini 2.5 Pro** — higher factfulness, higher
groundedness and goal-completion, at **~1/5 the cost**. This is a stronger result than
classification (where they tied).

**Where the gap came from (verified real, not a judge artifact):** Gemini's only two missed facts
were both on YoY-comparison rows (`car_net_profit_yoy`, `ncbfg_noi_yoy`). On both, Gemini gave the
correct figures and direction — even the absolute J$ change — but **did not state the YoY
percentage** the expected-fact asks for. GLM did (e.g. "+23.7%", "a decline of … approximately
5.54%"). Same judge, same bar; GLM was simply more complete on comparison reasoning.

**Both models handled the hard cases correctly:**
- The NCBFG row where net operating income *fell* ~6% (the one our draft had backwards) — **both got
  the direction right.**
- All **3 negative/out-of-scope prompts** (off-topic code, personalized buy/sell advice, price
  target) — **both correctly declined/redirected** (factfulness 1.00 on each).

GLM's one weakness: **tail latency** — its p95 (42.7 s, one slow fixture) is worse than Gemini's
(20.3 s). That's the only axis Gemini wins, and why both stay Pareto-optimal.

## Caveats (about the benchmark)

- **Small set (13).** Gemini's 0.923 = 2 missed facts of ~24; directional, not high-powered.
- **Single judge** (Claude Sonnet 4.6); neutral to both candidates, but one judge's calls.
- **Groundedness is ill-posed for the 3 negatives** (a correct refusal makes no grounded claims), so
  the low groundedness there (esp. Gemini's g1/g2) is expected, not a defect.
- Groundedness/goal_completion are computed per-sample (in the logs), not yet in the scorecard.

## Spend

~$0.35 total (candidates ~$0.15, judge ~$0.13, profiling ~$0.05). Trivial.

## Reproduce

```bash
export OPENROUTER_API_KEY=...   # from .env
inspect eval src/atsbench/tasks/rag_qa.py --model openrouter/z-ai/glm-5.2 \
  -M reasoning_enabled=true --temperature 0 --log-dir logs/rag
inspect eval src/atsbench/tasks/rag_qa.py --model openrouter/google/gemini-2.5-pro \
  -M reasoning_enabled=true --temperature 0 --log-dir logs/rag
atsbench report --logs logs/rag --workflow rag_qa
```
